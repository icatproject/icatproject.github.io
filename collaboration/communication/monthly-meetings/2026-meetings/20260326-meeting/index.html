ESRF

2 weeks ago, HOA wesbite had issues with ICAT. New article published, lots of visits caused ICAT to be slow. We don't know underlying cause, Alex made HOA static to not request to ICAT anymore. Need to investigate if it's too many connections, long queries etc. caused DB to crash.

Working on notifying users at end of embargo period.

Working on ESRF techniques ontology, updating to use W3ID. Pushing beamlines to use techniques - soon MX beamlines.

Alex working on CryoEM visualisation.

Working on aria2c download.

 

KP: Remember Alex asking about DB setup to look at scaling, reliability etc. across different ICAT facilities. Did anything come from that?

MB: HOA usecase we query on multiple investigations. VS dataportal usually 1 investigation. Maybe this caused the issue?

KP: general slowness hard to debug, we've had issues in past

MB: each page opens new anonymous session. Maybe try to cache anonymous session

RK: in theory caching session ID should be easy

ISIS

Nothing major. Metadata issues.

Talks about migrating backup system. DMF retiring.

Migrate current servers to new VM software.

DLS

Recently had a demo of the open data, DOI minting process together with Diamond legal team and some scientists. It went very well and got some useful feedback as well, which will need to be reconsidered or be worked on further. But issues not on the ICAT/DataGateway side. That's been the main thing.

Kevin has been looking at DB problems. No problems at the moment, but investigating preventative measures.
